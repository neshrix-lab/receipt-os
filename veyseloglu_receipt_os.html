Агент 2 · SKU-матчер
embeddings pgvector фоновый
embeddings · дёшево
«Кока-кола 0,5 ст/б» → SKU из номенклатуры 1С:УТ. Читает справочник 1С — не создаёт свой. Claude как арбитр только при неоднозначности среди топ-3 кандидатов. Не найдено → флаг оператору.
Вход
Строка наименования из накладной
Выход
SKU из 1С + % уверенности · или флаг «не найден»
Сверщик PO
детерминированный код читает 1С:УТ
без AI
Не AI. Простая функция: берёт заказ поставщику (PO) из 1С:УТ → сравнивает цены и количества строка за строкой → возвращает список расхождений с % отклонения. LLM здесь не нужен.
Вход
Позиции накладной + PO из 1С:УТ
Выход
OK / список расхождений с дельтой %
✓ confidence >95% · совпало → авто ⚠ SKU не найден или цена расходится → оператор
Авто-проводка
~94% накладных
Без участия оператора → сразу к записи в 1С
Оператор · 2 мин
~6% накладных
Исправить SKU вручную или подтвердить расхождение с PO
Проводчик 1С
COM / HTTP-сервис 1С:УТ API
без AI
Не AI. Детерминированный коннектор: создаёт приходную накладную в 1С:УТ. 1С сам обновляет складские остатки и передаёт данные в 1С:Бухгалтерию по стандартному механизму обмена.
Вход
Подтверждённые позиции + SKU + цены + склад
Выход
Номер документа в 1С · остатки обновлены
